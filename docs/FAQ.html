General Questions

What is the Bearsampp PowerShell module?

This is a module of the Bearsampp project that provides PowerShell 7+ with enhanced console features including Oh My Posh for beautiful prompt themes, PSReadLine for advanced command-line editing, and Nerd Fonts for rich icon support.

What are the main features?

  • Oh My Posh - Beautiful and customizable prompt themes
  • PSReadLine - Advanced command-line editing and history
  • Nerd Fonts - Rich icon and glyph support
  • Auto-completion - Intelligent command and path completion
  • Command History - Persistent command history across sessions
  • Syntax Highlighting - Color-coded command syntax

What versions of PowerShell are supported?

The module supports PowerShell 7.0 and later versions. You can build specific versions using the Gradle build system.

Is this compatible with Windows PowerShell 5.1?

This module is designed for PowerShell 7+. While some features may work with Windows PowerShell 5.1, full compatibility is not guaranteed.

Installation & Setup

What are the prerequisites for building this module?

You need:

  • Java JDK 8 or later
  • Gradle 8.0+
  • 7-Zip (required for .7z format)
  • Git

Do I need to install the Gradle Wrapper?

No, this project deliberately does not ship the Gradle Wrapper. You must install Gradle 8+ locally on your system.

Where can I download the prerequisites?

  • Java JDK: https://adoptium.net/
  • Gradle: https://gradle.org/
  • 7-Zip: https://www.7-zip.org/
  • Git: https://git-scm.com

How do I verify my build environment is ready?

Run the verification command:

gradle verify

This will check for Java, build.properties, required directories, and 7-Zip.

Building & Compilation

How do I build a specific version?

Use the release task with the version parameter:

# Non-interactive mode
gradle release -PbundleVersion=7.5.4

# Interactive mode (prompts for version selection)
gradle release

How do I build all available versions at once?

Use the releaseAll task:

gradle releaseAll

How can I see what versions are available to build?

List available versions with:

gradle listVersions

Where are the build outputs stored?

By default, builds are stored in:

C:/Bearsampp-build/tools/powershell/{release-date}/

You can customize this location in build.properties using the build.path property.

What files are generated during a build?

For each version, you get:

  • The main archive (.7z or .zip)
  • MD5 checksum (.md5)
  • SHA1 checksum (.sha1)
  • SHA256 checksum (.sha256)
  • SHA512 checksum (.sha512)

How do I change the archive format from 7z to zip?

Edit build.properties and change:

bundle.format = zip

What does the build process actually do?

The build process:

  1. Downloads PowerShell binaries from the modules-untouched repository
  2. Downloads dependencies (Oh My Posh, themes)
  3. Configures the PowerShell profile
  4. Packages everything into an archive
  5. Generates checksums for verification

Font & Display Issues

Why do I see boxes or question marks instead of icons?

You need to install and configure a Nerd Font. This module requires CaskaydiaCove NF (Cascadia Code Nerd Font) to display icons correctly.

How do I install the required font?

The CaskaydiaCove NF font should be installed system-wide via Bearsampp prerequisites. If not installed, download it from the Nerd Fonts website and install it in Windows.

How do I configure the font in Windows Terminal?

Add this to your Windows Terminal settings.json:

{
    "profiles": {
        "defaults": {
            "font": {
                "face": "CaskaydiaCove NF",
                "size": 10
            }
        }
    }
}

How do I configure the font in the classic Windows Console?

  1. Right-click the title bar → Properties
  2. Go to the Font tab
  3. Select "CaskaydiaCove NF" from the font list
  4. Click OK

The font is installed but icons still don't show. What's wrong?

Make sure:

  1. The font is installed system-wide (not just for your user)
  2. You've restarted your terminal after installing the font
  3. The font name is exactly "CaskaydiaCove NF" in your configuration
  4. Your terminal supports Unicode and UTF-8 encoding

Configuration

Where is the PowerShell profile located?

The profile is located at:

bin/powershell{version}/Microsoft.PowerShell_profile.ps1

How do I change the Oh My Posh theme?

Edit the profile file and change the theme path:

$ohMyPoshTheme = Join-Path $env:POSH_ROOT "themes\YOUR_THEME.omp.json"

Available themes are in the vendor/oh-my-posh/themes/ directory.

What themes are available?

The module includes all Oh My Posh themes. Popular ones include:

  • paradox (default)
  • agnoster
  • powerlevel10k_rainbow
  • atomic
  • jandedobbeleer

Browse the themes directory or visit https://ohmyposh.dev/docs/themes

How do I customize PSReadLine settings?

Edit the Set-PSReadLineOption calls in the profile to customize:

  • Colors and syntax highlighting
  • Key bindings
  • History behavior
  • Prediction settings
  • Edit mode (Windows, Emacs, Vi)

What environment variables does this module set?

  • POSH_ROOT - Path to Oh My Posh installation
  • POSH_THEMES_PATH - Path to Oh My Posh themes directory

How do I configure dependencies for a specific version?

Edit the deps.properties file in the version directory:

bin/powershell{version}/deps.properties

This file contains URLs for Oh My Posh and theme downloads.

Can I use a custom build output directory?

Yes, uncomment and set the build.path property in build.properties:

build.path = D:/MyCustomPath

Troubleshooting

The build fails with "7-Zip not found"

Install 7-Zip from https://www.7-zip.org/ and ensure it's in your system PATH, or install it to the default location (C:\Program Files\7-Zip\).

I get "Java not found" errors

Ensure Java JDK 8+ is installed and the JAVA_HOME environment variable is set correctly. Verify with:

java -version

Downloads fail during the build

Check your internet connection and verify that the URLs in releases.properties and deps.properties are accessible. The build system downloads from the modules-untouched repository.

The profile doesn't load when I start PowerShell

Ensure:

  1. PowerShell execution policy allows scripts: Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
  2. The profile is in the correct location
  3. There are no syntax errors in the profile (check with Test-Path $PROFILE)

Oh My Posh doesn't initialize

Check that:

  1. The posh-windows-amd64.exe file exists in the vendor directory
  2. The POSH_ROOT environment variable is set correctly
  3. The theme file exists at the specified path

Command history isn't working

PSReadLine manages command history. Check:

  1. The history file location: (Get-PSReadLineOption).HistorySavePath
  2. Ensure the directory is writable
  3. Verify PSReadLine is loaded: Get-Module PSReadLine

Advanced Usage

Can I build multiple versions with different configurations?

Yes, each version in the bin/ directory has its own deps.properties and bearsampp.conf files that can be configured independently.

How do I add a new PowerShell version?

  1. Create a new directory: bin/powershell{version}/
  2. Add deps.properties with dependency URLs
  3. Add bearsampp.conf with Bearsampp configuration
  4. Add the version mapping to releases.properties

Can I use this module outside of Bearsampp?

While designed for Bearsampp, the built archives contain a fully functional PowerShell setup that can be extracted and used independently.

How do I check dependencies configuration?

Run:

gradle checkDeps

This validates all deps.properties files.

Can I see what releases are available in modules-untouched?

Yes, run:

gradle listReleases

How do I get build information?

Run:

gradle info

This displays configuration details, available versions, and build settings.

Are downloads cached?

Yes, downloaded files are cached in the Gradle cache directory to speed up subsequent builds. Clean the cache with:

gradle clean

Can I customize the bundle release identifier?

Yes, edit the bundle.release property in build.properties:

bundle.release = 2025.11.13

How do I contribute or report issues?

https://github.com/bearsampp/bearsampp/issues
